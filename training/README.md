# Labelling Fields in Azure Document Intelligence Studio

This guide walks you through labelling fields in **Azure Document Intelligence Studio** using the filled sample PDFs generated by `generate_forms.py`. Labelled documents are used to train the custom extraction model defined in `config/fields.json`.

---

## Prerequisites

- Access to an **Azure Document Intelligence** resource (F0 or S0 tier)
- Sample PDFs uploaded to Azure Blob Storage (run `scripts/upload_to_blob.py`)
- `AZURE_BLOB_CONTAINER_SAS_URL` configured in your `.env`

---

## Step 1: Open Azure Document Intelligence Studio

1. Navigate to [Document Intelligence Studio](https://documentintelligence.ai.azure.com/studio)
2. Sign in with your Azure account
3. Select **Custom extraction model** → **Create a project**

## Step 4: Label Documents

1. Open each sample PDF in the Studio labeller
2. Draw bounding boxes around each field value
3. Assign the correct field name from Step 3
4. Repeat for all 12 sample documents

### Tips for RadioButton / SelectionMark fields

- For Yes/No radio buttons, label the **selected circle** as a `SelectionMark` field
- If both circles are visible, label only the one that is filled/selected
- Use field names like `AuditPartAQual`, `ECPIPaid`, etc.

### Tips for repeating member fields

- Each member occupies its own page (pages 8–13)
- Label the same field names on every page — the model learns the pattern across pages
- Leave unlabelled any page where the member fields are intentionally blank

---

## Step 5: Train the Model

Once at least 5 documents are fully labelled:

1. Click **Train** in the Studio, or run:
   ```bash
   python scripts/train_model.py
   ```
2. Wait for training to complete (typically 5–15 minutes)
3. The model ID is saved to `output/model_id.txt`

---

## Step 6: Test the Model

```bash
python scripts/test_model.py output/smsf_return_1members_1.pdf
```

Review confidence scores. Fields with confidence below 0.8 should be re-labelled with additional examples.

---

## Step 7: Extract from a Real Return

```bash
python scripts/extract_to_json.py path/to/real_return.pdf
```

Output is saved to `output/extracted_<filename>.json`, grouped by section.
